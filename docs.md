# VIBEBIOS: Техническая документация 🧠

В этом документе описана внутренняя архитектура WILIXBIOS и технические решения, примененные для обхода аппаратных ограничений x86.

## 1. Архитектура памяти ROM (linker.ld)
По стандарту x86, процессор при старте начинает выполнение инструкций с физического адреса `0xFFFFFFF0` (Вектор Сброса). QEMU мапит файл прошивки (`bios.bin`) в самый конец 4 ГБ оперативной памяти.

Наш BIOS весит ровно 128 КБ. Его карта памяти выглядит так:
* `0xFFFE0000` — Точка входа в 32-битный C-код (`bios_main`).
* `0xFFFFFF00` — 16-битный бутблок (настройка сегментов, GDT).
* `0xFFFFFFF0` — Reset Vector (содержит инструкцию дальнего прыжка назад на бутблок).

**Важно:** Утилита `objcopy` имеет свойство обрезать нули в конце бинарника. Чтобы файл весил ровно 131072 байта, в самый конец `boot.asm` добавлена инструкция `NOP` (`0x90`).

## 2. Инициализация и переход в 32 бита (boot.asm)
Один из самых сложных этапов — парсинг GDT из ROM в 16-битном режиме. 
Исторический баг процессоров x86 заключается в том, что 16-битная команда `lgdt` читает только 24 бита базового адреса, отбрасывая старший байт. Это приводит к `Triple Fault`, так как адрес `0xFFFE0000` превращается в `0x00FE0000`.

**Решение:** Мы применяем аппаратный префикс смены размера операнда `0x66` перед командой `lgdt`.
```nasm
db 0x66
lgdt [cs:bx] ; Процессор честно читает 32-битный адрес GDT из ROM
```

## 3. Инициализация PCI и High-Res Графики (bios.c)
Стандартный VGA адрес `0xA0000` ограничен размером в 64 КБ. Этого достаточно для разрешения 320x200 (64 000 байт), но недостаточно для 800x600 (480 000 байт). 
Для обхода лимита используется **LFB (Linear Frame Buffer)** видеокарты через шину PCI.

Процесс в `init_vga()`:
1. Обращение к конфигурационному порту PCI (`0xCF8` / `0xCFC`).
2. Принудительная запись физического адреса `0xE0000000` в `BAR0` видеокарты.
3. Включение бита *Memory Space Enable* в PCI Command Register (иначе северный мост заблокирует передачу пикселей).
4. Инициализация QEMU Bochs VBE (порты `0x01CE` / `0x01CF`) на разрешение 800x600x8bit.

*Примечание:* Указатель на видеопамять объявлен как `volatile u8* volatile LFB`. Это предотвращает агрессивную оптимизацию компилятора GCC (ключ `-Os`), заставляя его писать данные физически в оперативную память при отрисовке каждого пикселя.

## 4. Чтение HDD (ATA PIO)
BIOS не имеет доступа к `int 13h`. Мы написали собственный драйвер жесткого диска, который общается напрямую с контроллером IDE материнской платы.
Алгоритм в `read_sector_0()`:
1. Опрос порта `0x1F7` до тех пор, пока не спадет бит занятости (`BSY`).
2. Запись параметров в порты (Сектор 0, LBA режим, Master диск).
3. Подача команды чтения (`0x20`).
4. Ожидание флага готовности данных (`DRQ`).
5. Чтение 256 слов (512 байт) из порта данных `0x1F0` в адрес оперативной памяти `0x7C00`.

## 5. Взаимодействие с пользовательской ОС (os.asm)
При успешном нахождении сигнатуры загрузчика (`0x55 0xAA`), BIOS передает управление на `0x7C00` посредством вызова указателя на функцию. 

Поскольку BIOS работает в 32-битном режиме, пользовательская ОС также начинает работу в **32-битном режиме**. Ей не нужно настраивать GDT или переключаться из Real Mode. 
ОС может узнать текущий адрес видеопамяти LFB, просканировав шину PCI (BAR0), и сразу начать отрисовку своей графики или загрузку ядра на C.
